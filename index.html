计算机通信网

1、概述

(1)算机网络的定义：
自主计算机的互联集合。
(2)广播式网络与广播的区别：
广播式网络指的是主机间共享一条信道的主机网络；广播指的是一种发送数据的方式，一发全收。
(3)速率
a信号传播速率：电信号在固体介质中传播的速率，一般为2/3光速；
b数据传输速率：指数据发送的速度，每秒发射多少位（bps，bit per second)
(4)速率计算
信道无扰的理想情况，奈奎斯特定理：R = 2B *log2N，其中R为传输速率，B为带宽，N为信号状态数；
信道有干扰的情况，香农定理：C = B *log2(1 + S/N)，其中C为信道容量，B为带宽，S/N为信噪比。
(4)同步异步
同步用于信道质量较好的环境，异步用于信道质量较差的环境。
(5)网络分层模型
上下层之间的是“服务”，同一层之间的是“协议”。
OSI七层模型：应用层->表示层->会晤层->传输层->网络层->数据链路层->物理层。
a、物理层提供bit流传输的服务。
b、数据链路层通过成帧、帧差错检测、处理、流量控制等实现线路两端正确通信。
c、网络层进行路由计算、选择；网络互联和组网。
d、传输层进行网络分流、复用、流量控制等。
e、会话层进行通话控制，决定何时、何人可以通话。
f、表示层为应用层提供语法语义的表示，对数据进行加密压缩等。
g、应用层就是用户操作的层次，如网上购物、文件传输等。
*注意：服务一般用于上下层之间的描述；协议则是对应通信两端面同层次之间的描述。
(7)简单通信存在问题
不够可靠、传输距离不够远、效率不高、不同网络设备之间不能互联。
(8)网络互连
通过直接互连（网络接口上实现转换）；间接互连（网关处实现转换）两种方式实现

基于klonenet平台的kruskal算法实现

kruskal算法是是应用于求出最小生成树问题的经典算法。该次作业要求如下：
1、使用python获取给定json文件中编制好的网络信息，包括主机mac地址、交换机dpid、带宽、传输延时等；
2、选取适宜的数据结构储存读取到的网络信息；
3、根据数据结构实现kruskal算法，获取最小生成树路径。

最小生成树

对于一个连通无向图G=(V, E)，我们希望找到一种连接方式使得所有结点能够连接起来并且拥有最小的边权重之和。如图中红线就是该网络的最小生成树

使用贪心策略的解决办法

知晓贪心的读者应该已经大概知道如何去找到这个最小生成树了，接下来展示课本《算法导论》中提炼的伪码实现：
A = ∅
while A does not from the spanning tree
find an edge(u, v) that is safe for A
A = A∪{(u, v)}
return A
其中，每次循环A都是某棵最小生成树的子集；而对于A安全的边指的是该条边加入到集合A后不会破坏A的循环不变式。

kruskal算法

kruskal算法便是上文伪代码的一种细化方式，我们很容易地看到贪心策略的解决办法其难点在于安全边的确定。kruskal算法的解决方式是每次都选取边权最小的边来加入集合A。伪代码如下：
A = ∅
for each vertex v ∈ G.V
MAKE-SET(v)
sort the edges of G.E into nondecreasing order by weight w
for each edges e(u, v) in G.E
if FIND-SET(u) ≠ FIND-SET(v)
A = A∪{(u, v)}
UNION(u, v)
return A
也就是说，kruskal算法将图的每一个结点都独立为一个集合（MAKE-SET），再根据边权由小到大进行排序，遍历边的集合时判断边的两个顶点是否属于同一集合（同一棵树），不是则将这条边加入集合A，再将u，v所在集合合并（UNION）。如若u，v顶点属于同一集合依旧将(u,v)边加入A的话那A中集合就会成环，最后结果就不是最小生成树了。
